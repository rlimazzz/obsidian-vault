## Modelo cliente-servidor

Arquitetura de rede onde os dispositivos clientes solicitam recursos ou serviços, e computadores servidores os processam e fornecem.
## O que é computação em nuvem?

É a entrega sob demanda de recursos de TI pela internet com preços de pagamento conforme o uso.

Os benefícios da AWS são :
	Parar de tentar adivinhar a capacidade necessária
	Flexibilidade para operar no mundo todo sem a necessidade de mais investimentos
	Não existe custos de manutenção
	Alta disponibilidade 
	Tolerância a falhas

## Modelo de Responsabilidade Compartilhada

A AWS é responsável pela segurança da nuvem, bem com atualização de softwares de computação, armazenamento, banco de dados e redes. (Segurança DA Nuvem)
O cliente é responsável pela criptografia de dados, pelos dados, etc. (Segurança NA Nuvem)

## Avaliação

Uma Região é um local físico em todo o mundo que contém vários data centers isolados. Uma Zona de Disponibilidade consiste em um ou mais data centers discretos, cada um com alimentação, rede e conectividade independentes.

A infraestrutura global normalmente envolve a distribuição de recursos em vários data centers em diferentes localizações geográficas. Ao ter sistemas redundantes em vários locais, a infraestrutura global aumenta a tolerância a falhas. Se um componente falhar, outros podem assumir o controle para minimizar o tempo de inatividade e ajudar a garantir a alta disponibilidade.

Facilidade de uso, conformidade e economia de custos são benefícios que podem ser obtidos com o uso de recursos em nuvem. No entanto, nesse caso, o foco é tornar os recursos altamente disponíveis para usuários globais.

No modelo de responsabilidade compartilhada da AWS, o cliente é responsável pelo gerenciamento de patches do sistema operacional. A AWS fornece a infraestrutura subjacente, mas o cliente deve garantir que seus sistemas operacionais estejam atualizados com os patches de segurança mais recentes. O cliente também é responsável por criptografar os dados do lado do cliente. A AWS fornece vários serviços para ajudar na criptografia, mas é responsabilidade do cliente implementar e gerenciar esses serviços.

A segurança física dos data centers e a manutenção do hardware são responsabilidades da AWS. A atualização de software para serviços de computação, rede, armazenamento e banco de dados também é uma responsabilidade da AWS.
